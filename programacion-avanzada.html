Programación Orientada a Objetos (POO)

La Programación Orientada a Objetos (POO) es un paradigma de programación fundamental en el desarrollo de software moderno. Se basa en el concepto de "objetos", que son entidades que combinan estado (datos) y comportamiento (métodos).

Conceptos Clave de OOP

  • Clases: Plantillas o planos para crear objetos. Definen las propiedades (atributos) y comportamientos (métodos) que tendrán los objetos de ese tipo.
  • Objetos: Instancias de una clase. Cada objeto tiene su propio estado y puede realizar las acciones definidas por su clase.
  • Herencia: Mecanismo que permite crear nuevas clases (subclases o clases derivadas) basadas en clases existentes (superclases o clases base). Facilita la reutilización de código y la creación de jerarquías de clases.
  • Polimorfismo: Capacidad de que objetos de diferentes clases respondan a la misma llamada de método de manera diferente. Permite escribir código más genérico y flexible.
  • Encapsulamiento: Ocultar los detalles internos de un objeto y exponer solo una interfaz para interactuar con él. Protege la integridad del estado interno del objeto y reduce la complejidad.

Ejemplo de Código Java (OOP)

Ejemplo simple de una clase `Animal` en Java que demuestra conceptos básicos de OOP como clases, objetos, herencia y polimorfismo.


            // Clase base Animal
            class Animal {
              private String nombre;

              public Animal(String nombre) {
                this.nombre = nombre;
              }

              public String getNombre() {
                return nombre;
              }

              public void hacerSonido() {
                System.out.println("Sonido genérico de animal");
              }
            }

            // Subclase Perro que hereda de Animal
            class Perro extends Animal {
              public Perro(String nombre) {
                super(nombre);
              }

              @Override
              public void hacerSonido() {
                System.out.println("Guau guau!");
              }
            }

            public class Main {
              public static void main(String[] args) {
                Animal animalGenerico = new Animal("Animal");
                Perro miPerro = new Perro("Buddy");

                animalGenerico.hacerSonido(); // Imprime "Sonido genérico de animal"
                miPerro.hacerSonido();        // Imprime "Guau guau!"

                System.out.println(animalGenerico.getNombre()); // Imprime "Animal"
                System.out.println(miPerro.getNombre());        // Imprime "Buddy"
              }
            }

Estructuras de Datos Avanzadas

Las estructuras de datos son formas especializadas de organizar y almacenar datos en una computadora para que puedan ser utilizados y procesados de manera eficiente. Las estructuras de datos avanzadas permiten manejar grandes volúmenes de datos y optimizar algoritmos para tareas complejas.

Listas Enlazadas

Una lista enlazada es una secuencia de nodos, donde cada nodo contiene datos y un enlace (puntero) al siguiente nodo en la secuencia. A diferencia de los arrays, las listas enlazadas no tienen un tamaño fijo y pueden crecer o decrecer dinámicamente.

  • Tipos de Listas Enlazadas: Simplemente enlazadas (enlace a un nodo siguiente), doblemente enlazadas (enlace a nodo siguiente y anterior), circulares (el último nodo apunta al primero).
  • Ventajas: Inserción y eliminación eficiente de elementos en cualquier posición, tamaño dinámico.
  • Desventajas: Acceso secuencial (no se puede acceder directamente a un nodo por índice), mayor uso de memoria debido a los punteros.

Árboles

Un árbol es una estructura de datos jerárquica compuesta por nodos. El nodo superior se llama raíz, y cada nodo puede tener cero o más nodos hijos. Los árboles se utilizan para representar relaciones jerárquicas y para búsquedas eficientes.

  • Tipos de Árboles: Árboles binarios (cada nodo tiene como máximo dos hijos), árboles AVL y árboles rojinegros (árboles binarios de búsqueda auto-balanceados), árboles B (optimizados para acceso a disco).
  • Aplicaciones: Sistemas de archivos, índices de bases de datos, representación de jerarquías, algoritmos de búsqueda y ordenamiento.
  • Ejemplo de Árbol Binario: 
    
                      Raíz (10)

                 (5)        (15)
                /   \      /   \
             (2)    (7)  (12)   (20)

Grafos

Un grafo es una estructura de datos no lineal que consiste en un conjunto de nodos (vértices) y un conjunto de aristas (conexiones) que relacionan pares de nodos. Los grafos son muy versátiles y pueden representar relaciones complejas entre objetos.

  • Tipos de Grafos: Dirigidos (aristas con dirección), no dirigidos (aristas sin dirección), ponderados (aristas con pesos o costos), no ponderados.
  • Aplicaciones: Redes sociales, mapas y navegación, análisis de redes, algoritmos de rutas más cortas, planificación de proyectos.
